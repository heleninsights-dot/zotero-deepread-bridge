# Zotero Deep Read Bridge

Bridge from Zotero collections to AI deep-reading pipelines. Three things it can do:

1. **List** Zotero collections with IDs and full paths.
2. **Export** every PDF in a collection into a normal folder with a traceable manifest.
3. **Deep Read** — export + automatically run `phd-deepread batch` to generate literature notes and canvases.

The tool is read-only against the local Zotero SQLite database. No API keys, no cloud.

## When to activate

The user mentions Zotero collections and wants to:

- "list my Zotero collections" / "what collections do I have in Zotero?"
- "export my collection" / "pull PDFs from Zotero into a folder"
- "batch deep-read this collection" / "process all papers in my Zotero folder"
- "deepread my Zotero collection" / "run phd-deepread on a Zotero collection"
- They refer to a Zotero collection by name and want to do something with its PDFs

If the user just wants to read a single PDF, use the `phd-deepread` skill instead. This skill is for **collection-level** operations.

## How to run it

You drive the `zotero-deepread-bridge` on the user's behalf. The user does not type these commands.

### Finding the tool

Try the pip-installed command first. If it's missing, install it silently on the
user's behalf — this is a one-time thing:

```bash
zotero-deepread --help 2>/dev/null || \
  pip install --quiet git+https://github.com/heleninsights-dot/zotero-deepread-bridge.git
```

After that, `zotero-deepread` is available for all subsequent commands.

### List collections

```bash
zotero-deepread list
```

Show the output to the user as-is — they need the collection IDs/names to pick what to export next.

### Export PDFs (dry-run first, always)

Always start with a dry-run so the user knows what they're getting:

```bash
zotero-deepread export "COLLECTION" -o "OUTPUT_DIR" --dry-run
```

Show the user: collection name, PDF count, output folder. Then ask for confirmation before running without `--dry-run`.

```bash
zotero-deepread export "COLLECTION" -o "OUTPUT_DIR"
```

### Export + Deep Read

Dry-run first, same as export:

```bash
zotero-deepread deepread "COLLECTION" \
  --pdf-output "PDF_OUTPUT_DIR" \
  --deepread-output "DEEPREAD_OUTPUT_DIR" \
  --dry-run
```

Then run without `--dry-run`. After export, `phd-deepread batch` runs automatically. Canvases are generated by default (`--create-canvases` is on).

### Key options worth remembering

- `--mode symlink` or `--mode hardlink` — save disk space instead of copying
- `--no-include-subcollections` — only the selected collection, not its children
- `--db ~/custom/path/zotero.sqlite` — non-standard Zotero database location
- `--linked-attachments-base` — needed if the user uses Zotero's linked-attachments feature

## Recommended vault layout

Suggest this layout inside the user's Obsidian vault:

```
Research/
  Zotero PDF Inbox/         ← exported PDFs live here
    YourCollection/
      [ABC123] Smith - 2023 - Functional analysis.pdf
      zotero-pdf-manifest.csv
      zotero-pdf-manifest.json
  Deep Read Output/           ← generated notes and canvases
    YourCollection/
      markdown_output/
      structured_literature_notes/
      *.canvas
```

The manifest files (`zotero-pdf-manifest.csv` + `.json`) trace every PDF back to its Zotero item — advise the user to keep them.

## After export — what to tell the user

- How many PDFs were exported
- Where they went
- Whether any PDFs were missing (recorded in the manifest as `missing-source`)
- If they ran `deepread`, point them to the generated notes and canvases
- Remind them to open the canvases in Obsidian to refine the nodes

## Troubleshooting

- **"No collection matched 'X'"** — run `zotero-deepread list` and pick the exact ID or full path.
- **"Collection name is ambiguous"** — multiple collections share the name. Show the user the choices and ask them to pick by ID.
- **"No PDFs were exported"** — the collection may have no PDF attachments. Suggest checking in Zotero directly.
- **Missing source PDFs** — Zotero's storage may be on a different path. Ask about `--zotero-data` or `--linked-attachments-base`.
- **"phd-deepread: command not found"** — the phd-deepread CLI isn't installed. The export still worked; the user can run deep-reading separately or install phd-deepread.
